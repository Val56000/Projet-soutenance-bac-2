Bienvenue dans le projet Python



I. Présentation du contexte

L'association Défis souhaite mettre en place un programme permettant de merger (fusionner) des fichiers conflictuel.



1. Les cibles du programme

Les utilisateurs du programme seront :
- L'association Défis et ses partenaires
Chacun de ces utilisateurs utilisent le serveur NextCloud pour accéder à des fichiers partagés.



2. NextCloud, c'est quoi ?

NextCloud est un logiciel libre de site d'hébergement de fichiers et une plateforme de collaboration.
Grâce à son architecture ouverte, il est compatible avec l’ensemble des navigateurs et systèmes d’exploitation.

NextCloud propose de multiples fonctionnalités : synchronisation des serveurs internes, partage automatique des fichiers,
stockage sécurisé et chiffré de documents, etc. Il est possible de l’associer à certaines applications comme le calendrier,
la messagerie, la gestion des tâches et des contacts, l’édition en ligne de fichiers, ou encore la prise de notes.



3. Qu'est ce qu'un fichier conflictuel ?

Le principe du conflit est simple: deux utilisateurs travaillent en même temps sur un même fichier.
Pour conserver les 2 versions et ne pas perdre de données une version en conflit se génère.

Plus précisement, quand un utilisateur ouvre un fichier Office, il est verrouillé par le Client de Synchronisation du serveur.
Donc lorsqu'un autre collaborateur va ouvrir ce fichier il ne pourra le faire qu'en Lecture seule.

La seule manière pour que deux utilisateurs puissent effectuer une modification en simultanée est
que l'un soit connecté au serveur NextCloud et que l'autre soit en mode Hors-connexion (pas de connexion internet).

Prenons un exemple : Un premier utilisateur, connecté au serveur NextCloud, va enregistrer une modification
sur un fichier qui sera synchronisé sur le serveur.
Au même moment, un second utilisateur (en mode Hors-connexion) va également modifier ce fichier.
C'est une fois que le second utilisateur se connecte au serveur NextCloud, que le client de synchronisation
fait le lien entre les deux fichiers et détecte une modification simultanée. Il va alors considérer
ce deuxième fichier comme un fichier conflictuel et ne sera pas synchronisé sur le serveur


4. Qu'est ce qu'un client de synchronisation ?

La synchronisation est une action ou une étape. Elle consiste à dupliquer les données stockées en local sur un serveur dans le but
de les faire correspondre au fil du temps. Ainsi, les informations sont constamment mises à jour.

Quant au client de synchronisation, c'est un logiciel permettant de synchroniser automatiquement entre plusieurs ordinateurs
les fichiers de toute une arborescence de dossiers.




II. Description fonctionnelle et technique du programme

1. Le cahier des charges

Le besoin : créer un programme permettant de merger des fichiers en conflit
Les cibles : l'association Défis et ses partenaires
Langage de programmation utilisé : Python
IDE utilisé : Visual Studio Code pour réaliser le programme en local


Missions principales :
1. Localiser le dossier où seront manipulés les fichiers
2. Mettre tous les fichiers de ce dossier dans une liste
3. Chercher dans la liste de fichiers, les noms des fichiers comprenant "conflicted copy"
4. Créer un message d'alerte pour l'utilisateur afin de le prévenir d'un conflit existant
5. Créer une boîte de dialogue demandant à l'utilisateur s'il veut merger des fichiers
6. Trouver une solution pour sélectionner le fichier source et le fichier en conflit
7. Créer un nouveau fichier ayant le contenu du fichier source et le contenu du fichier en conflit grâce au merge
8. Créer un élément séparateur dans le troisième fichier pour indiquer quel est le contenu 1 et quel est le contenu 2
9. Vérifier que le nouveau fichier sera placé dans le répertoire souhaité

Missions annexes :
10. Pouvoir merger d'autres format de fichier en plus de l'extension .txt (expl docx, etc ..)
11. Trouver une solution pour que le contenu du fichier source et le contenu du fichier en conflit s'implante bien dans le troisième fichier (pas de doublons)
12. Mise en pré-prod

Temps estimé : 3 semaines pour les missions principales (du 29/11/2021 au 17/12/2021)




2. Le langage Python


Python est un langage de programmation puissant et facile à apprendre. Il est idéal pour l'écriture de scripts et le développement
rapide d'applications dans de nombreux domaines et sur la plupart des plateformes.

Les avantages d'utiliser le langage Python :
- Bibliothèques étendues et très complètes ce qui permet de faire énormément de choses
- Amélioration de la productivité (La simplicité du langage et les bibliothèques étendues permettent d'écrire moins et de faire plus de choses
- Lisible (Python n'est pas un langage très verbeux, pas d'accolades pour définir les blocs et l'indentation est obligatoire)
- Langage interprété (les instructions sont exécutées une par une, le débogage est plus facile que dans les langues compilées)
